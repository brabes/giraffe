Steps to create a release archive.

The version is specific by VERSION.
REPODIR should be set to the root directory of the Git repo.

  VERSION=1.0a4
  REPODIR=~/SML/Giraffe/devel


Create a temporary directory for the release image

  RELEASEDIR=/tmp/release/
  mkdir -p ${RELEASEDIR}


Copy source code to the release directory.  Either:

 A. To create a formal release from a Git tag, extract the source code
    with Git.  VERSION is used as the Git tag name.

      cd ${REPODIR}
      git archive ${VERSION} giraffe | tar -x -C ${RELEASEDIR}


 B. To create an informal release from the working directory, take a copy
    after cleaning up any generated files:

      cd ${REPODIR}/giraffe
      make distclean
      cp -dpr ${REPODIR}/giraffe ${RELEASEDIR}


Remove non-release components:

  cd ${RELEASEDIR}/giraffe
  rm -rf codegen codegen.x86 graveyard doc/design gir
  rm -rf doc/user_guide  # just for now
  rm -rf auto/girepository-2.0
  rm src/polyml/giraffe-girepository-2.0.c


Make TAR file

  cd ${RELEASEDIR}
  tar --owner=root --group=root -czf giraffe-${VERSION}.tar.gz giraffe
